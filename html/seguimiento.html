Ejecución y Seguimiento

Descubre cómo definimos los procedimientos, variables e instrumentos clave para garantizar el éxito del proyecto.

Explorar Procedimientos

Procedimientos de Seguimiento

Para garantizar un control efectivo durante la ejecución del proyecto, hemos definido los siguientes procedimientos:

  • Reuniones semanales para evaluar el progreso y resolver bloqueos.
  • Uso de herramientas de gestión de proyectos como Trello para asignar tareas y monitorear su estado.
  • Revisión de hitos clave al final de cada sprint.
  • Documentación continua de avances y problemas encontrados.

Selección de Variables

Las variables seleccionadas para el seguimiento del proyecto son fundamentales para medir el éxito y el progreso:

  • Tiempo: Control del cumplimiento de plazos establecidos para cada tarea.
  • Calidad: Verificación de que los entregables cumplen con los estándares definidos.
  • Recursos: Supervisión del uso eficiente de los recursos asignados.
  • Riesgos: Identificación y mitigación de posibles problemas durante la ejecución.

Instrumentos Empleados

Los instrumentos seleccionados para el seguimiento del proyecto son los siguientes:

  • Herramientas de gestión: Uso de Trello y GitHub para la planificación y seguimiento de tareas.
  • Reportes: Generación de informes semanales para evaluar el progreso.
  • Reuniones: Sesiones de revisión con el equipo para discutir avances y problemas.
  • Indicadores clave: Uso de métricas como porcentaje de tareas completadas y desviaciones de tiempo.

Organización y planificación del trabajo en equipo

El equipo de desarrollo estuvo compuesto por cuatro integrantes, quienes distribuyeron las tareas en forma incremental para construir la aplicación de manera progresiva. Siguiendo un enfoque iterativo e incremental, el proyecto se descompuso en etapas cortas (iteraciones) en las que cada ciclo añadía nueva funcionalidad al producto. De este modo, se establecieron objetivos parciales claros y entregas continuas, lo que facilitó coordinar el esfuerzo de todos los miembros y obtener avances constantes. Cada iteración incluía las tareas necesarias para cumplir con los objetivos planificados en ese periodo, garantizando que al final de cada ciclo el producto tuviera un estado funcional mejorado

Herramientas de gestión y comunicación

Para asegurar una gestión colaborativa de las tareas, se emplearon diversas herramientas especializadas. Trello sirvió como tablero Kanban para visualizar y asignar las tareas pendientes, en curso y finalizadas. Esta plataforma de gestión de proyectos facilita la organización de las tareas de forma visual e intuitiva, permitiendo que varios usuarios colaboren en un mismo proyecto. Del mismo modo, GitHub se utilizó como sistema de control de versiones para el código fuente de la aplicación. El control de versiones registra los cambios realizados y facilita el trabajo en equipo, ya que todos los desarrolladores pueden integrar sus modificaciones de forma coordinada. Además, la aplicación se desarrolló mediante módulos personalizados de Odoo, un ERP de código abierto que permitió extender la funcionalidad estándar según los requisitos del proyecto. Para la comunicación interna y la coordinación en tiempo real se empleó Discord, plataforma de mensajería y voz que ofrece canales organizados para distintos temas. Gracias a Discord, el equipo mantuvo un flujo de comunicación estructurado y accesible, con espacios dedicados a debates técnicos, dudas puntuales o coordinación de tareas. Esta herramienta aportó orden a la comunicación del equipo, dando soporte a la modalidad de trabajo colaborativo aún en entornos remotos o distribuidos.

Seguimiento y evaluación del avance

El progreso del proyecto se monitoreó mediante reuniones de seguimiento periódicas. Cada semana, típicamente los lunes, se celebraba una reunión breve de unos 15 minutos en la que cada integrante reportaba el estado de sus tareas. Esta dinámica se asemeja a las ceremonias de trabajo ágil descritas en la literatura, cuyo objetivo principal es mejorar la comunicación y alinear al equipo en cuanto a los próximos pasos. En cada sesión se revisaba qué tareas se habían completado, cuáles estaban en curso y qué bloqueos surgían, permitiendo al equipo adaptar prioridades o redirigir esfuerzos de inmediato. Al concluir cada reunión semanal se evaluaba el grado de cumplimiento de los objetivos previstos, comparando el avance real con lo planificado para esa iteración. Este procedimiento de evaluación continua permitió detectar a tiempo desvíos o retrasos, y ajustar la planificación de las siguientes semanas en consecuencia.

Dificultades organizativas y soluciones

A lo largo del proyecto surgieron desafíos relacionados con la gestión del tiempo y la organización interna. En particular, al principio las reuniones semanales de corta duración resultaron insuficientes para resolver algunos temas complejos o urgentes. Para solventar esto, el equipo acordó intensificar la comunicación en momentos críticos: se amplió la duración de ciertas reuniones y se estructuraron con agendas claras y puntos a tratar. De esta manera, cada encuentro fue más productivo y centrado en la resolución de problemas específicos. Los ajustes en la dinámica de trabajo —como aumentar la frecuencia de discusión de puntos clave y documentar los acuerdos de cada reunión— permitieron mejorar la coordinación temporal del equipo. En conjunto, estas medidas garantizaron que los problemas de organización no afectaran el avance del proyecto y que las tareas continuaran ejecutándose de forma ordenada.
